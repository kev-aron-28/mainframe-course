## Cual es la estructura de un programa cobol?
Un program en cobol se divide en 4 divisiones principales:

- IDENTIFICATION DIVISION: Informacion del programa
- ENVIRONMENT DIVISION: Configuracion del entorno (archivos)
- DATA DIVISION: Declaracion de variables y estructuras
- PROCEDURE DIVISION: Logica del programa

## Que es working storage?
 Seccion dentro de DATA DIVISION donde se declaran variables globales que mantienen su valor durant toda la ejecucion del programa

## Diferencia entre working storage y local storage?
working -> Retiene su valor durante toda su ejecucion
local-storage -> Se reinicializa cada vez que se ejecuta el programa

## Que es un copybook?
Es un archivo externo reutilizable que contiene definiciones de datos o estructuras y se incluye con la sentencia COPY

## Que significa PIC S9(8) COMP?

## Que es FD?
File Descrition: se usa dentro de DATA DIVISION, FILE SECTION para definir la estructura de un registro de un archivo

## Que es FILE STATUS?
Variable que guarda el estado de una operacion de archivo
00 -> Exito
10 -> Fin de archivo

## Diferencia entre OPEN INPUT, OUTPUT, I-O
INPUT, solo lectura
OUTPUT, solo escritura
I-O, lectura y escritura

## QUe pasa si lees mas alla de EOF?
Se genera codigo de fin de archivo (FILE STATUS 10)

## Que es un VSAM KSDS?
VSAM (Virtual Storage Access Method) es un tipo de archivo indexado
KSDS(Key sequenced data set) permite acceso por clave

## Diferencia entre PERFORM y GO TO?
PERFORM -> EJECUTA UN BLOQUE ESTRUCTURADO 
GO TO -> Salta a una etiqueta

## Que es un perform until?
Bucle que se ejectua hasta que una condicion sea verdera

## Que es un ABEND?
